Configuration can be done through options or by adding or customizing styles, components or other existing implementations.

## Options

Configure the `cwd`, `pattern` and other options like `rehype`/`remark` plugins.

<Info>
  For convenience, you can re-use the fields in `package.json`, if they are
  defined.
</Info>

```ts {4-5,18-26,11-12} showLineNumbers
export type MDXServiceBaseOptions<T extends UnknownFrontMatter> = {
  cwd: string;
  mdxBundlerOptions?: MDXBundlerOptions;
  sortProvider?: SortProvider<MDXServiceReturnType<T>>;
  tocPlugin?: (headings: DocHeading[]) => Plugin;
  filePattern?: string;
  fileProvider?: () => Promise<SourceFileType[]>;
};
```

## Bring your own!

Use-case specific configurations and implementations are kept to a minimum. Adding additional solutions or extending the existing ones is recommended.
